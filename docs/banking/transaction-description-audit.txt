ALTA BANK — CUSTOMER TRANSACTION DESCRIPTION AUDIT
==================================================

Scope: Every customer-visible transaction description written to bankTransaction,
altaCardTransaction, and customer-facing loan ledger entries, plus display-layer type
labels that appear alongside descriptions on statements and activity feeds.

Method: Code audit of description writers in src/server/, src/lib/bank/, and display
mappers. No logic or accounting changes proposed — copy only.

Surfaces where customers see descriptions:
- Bank account activity & PDF statements (tx.description + separate typeLabel column)
- Alta Card history & card statements (description + type column via altaCardTransactionLabel)
- Loan detail ledger (description + typeLabel)
- Account dashboard recentActivity (prefixes type before description — creates redundancy)

CROSS-CUTTING ISSUES
--------------------

1. DUPLICATE / REDUNDANT LABELING
   Customers see both typeLabel and description. Example on interest:
   - Type column: Interest Credit
   - Description: Interest credit (automatic) or Manual Interest Credit — All Categories
   - Dashboard recentActivity: Interest credit · Manual Interest Credit — All Categories

   Recommendation: Descriptions should carry the narrative; type labels stay short
   category tags. Avoid repeating "interest credit" in both.

2. INTERNAL IDs IN CUSTOMER COPY
   Loan bank transactions use truncated UUIDs (Loan payment · cmqvyaf8). Customers
   cannot interpret these.

3. OPERATOR / ADMIN / MANUAL LANGUAGE
   Any string containing Operator, Admin, Manual, Multi-Category, All Categories, or
   Batch violates the stated philosophy.

4. APPROVED DEPOSITS/WITHDRAWALS NEVER UPDATE DESCRIPTION
   Pending requests are created as "Deposit request" / "Withdrawal request". Approval
   updates status only — customers see "Deposit request" on completed deposits forever.

   Source: src/server/bank.service.ts — approveDeposit, approveWithdrawal

5. ALTA PAY PREFIX PARSING DEPENDENCY
   listUserAltaPaySent / listCompanyAltaPayReceived strip fixed prefixes:
   - Alta Pay business payment to
   - Alta Pay business payment from

   Any copy change to bank-leg Alta Pay descriptions must update parsers in
   src/server/alta-pay.service.ts simultaneously, or Alta Pay history panels break.

6. TYPE LABEL INCONSISTENCIES (DISPLAY LAYER)

   Code              Current label     Issue
   interest_credit   Interest Credit   Title case
   recentActivity    Interest credit   Sentence case — inconsistent
   alta_pay (card)   Alta Pay Purchase OK
   cash_advance      Cash Advance      Title case
   loan_payment      Loan payment      Sentence case

   Recommendation: Standardize display labels — Title Case for product names
   (Alta Pay, Alta Card, Money Market), sentence case for generic actions
   (Loan payment, Interest payment) — pick one system and apply everywhere.

END OF AUDIT
============

This audit covers every transaction type currently written to customer-visible ledger
tables in the codebase. The highest-impact fixes are manual interest copy, post-approval
deposit descriptions, Alta Pay bank-leg wording, and removing operator/admin/manual
language from anything that appears on statements or activity feeds.
