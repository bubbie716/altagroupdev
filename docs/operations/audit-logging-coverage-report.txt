Alta Audit Logging Coverage Report
===================================
Scope: Alta web app (altaweb) + Alta Companion Discord Bot (altabot)
Date: July 3, 2026

Architecture
------------
All DB audit rows flow through writeAuditLog() -> AuditLog table -> optional Discord
staff channel via audit-log-discord-bridge.ts. The bot never writes audit rows
directly; it calls shared altaweb services with metadata.source: "discord_bot".

Overall assessment: Money-movement and operator actions are well covered (~155
distinct action codes). Gaps cluster around failed attempts, attribution/source
accuracy, naming inconsistency, and high-volume automated events that inflate the
court record without adding legal value.

5. RECOMMENDED METADATA FIELDS
================================================================================

Required fields in metadata (every audit row):
  source        "website" | "discord_bot" | "cron" | "system" | "operator"
  severity      "info" | "warning" | "critical"
  timestamp     ISO 8601 (already auto-injected)
  correlationId string — tie multi-row events (accept -> activate -> tag)
  ipAddress     string? — web customer actions where available
  userAgent     string? — web customer actions

Domain-specific fields:
  Banking       amount, currency, referenceCode, transactionId, fromAccountId,
                toAccountId, transferScope, failureReason
  Alta Pay      fundingSource (bank_account | alta_card), cardId, recipientType,
                recipientId
  Credit        applicationId, loanId, cardId, previousStatus, newStatus,
                previousLimit, newLimit
  Operator      reason, reviewNote, ticketId, customerNotificationSent,
                silentNotification
  Deal rooms    dealRoomId, dealRoomType, messageId, discordChannelId,
                syncDirection
  Cron          jobKey, runId, durationMs, recordsProcessed, failureCount
  Failures      errorCode, errorMessage (sanitized, no PII/secrets)
  Bot           discordUserId, guildId, interactionId

Court-ready description template:
  {Actor} {verb} {entity} for {target} — {amount/reference} — {reason if operator}

6. STAFF DISCORD AUDIT MESSAGES
================================================================================

Should send Discord (actionable / high-signal):
  - Requires staff action: deposit/withdrawal submitted, review flags, sync failed,
    credit desk state change
  - Money movement: transfers, Alta Pay, loan/card payments, manual adjustments, fees
  - Access/security: account frozen, tag granted/revoked, permission denied (when
    added), maintenance enabled
  - Credit decisions: loan/card application approved/denied, review outcomes
  - Customer comms failure: notification delivery failed (when added) — critical
  - Bot failures on sensitive actions: transfer/pay/deposit submit failures (when added)

Should NOT send Discord (DB only):
  - Automated CRM: relationship profile/score/tier/recommendation (already skipped)
  - Message sync: deal room Discord<->web sync (already skipped)
  - Thread messages: loan/card application thread messages (already skipped)
  - Infrastructure: channel created/linked/locked (already skipped)
  - Batch automation: per-account interest accrual — should be added to skip list;
    keep batch summary only
  - Read-only ops: previews, readiness views (already skipped)

Fix attribution bugs for Discord:
  - ALTA_PRIVATE_ACTIVATED and PRIVATE_BANKING_CLIENT_MARKED after bot accept must
    carry source: "discord_bot"
  - Add Discord labels for ~80 actions currently using fallback humanize

8. PRIORITY ORDER FOR FIXES
================================================================================

P0 — Legal / money integrity (do first)
  1. Audit bot banking failures — deposit/withdrawal/transfer/Alta Pay blocked or
     errored attempts with failureReason, source: discord_bot
  2. Audit Discord convenience fee — BANK_FEE_POSTED or dedicated fee action when
     collectDiscordTransferFee() runs
  3. Add BANK_TRANSFER_*_FAILED and BANK_ALTA_PAY_FAILED on web + bot error paths
  4. Fix ALTA_PRIVATE_ACTIVATED source attribution on bot accept path
  5. Add fundingSource metadata (or split ALTA_CARD_ALTA_PAY_CHARGED) for
     card-funded Alta Pay

P1 — Court readiness / consistency
  6. Standardize denial action names — migrate BANK_PAYMENT_BLOCKED ->
     BANK_DEPOSIT_DENIED / BANK_WITHDRAWAL_DENIED (or alias in exports)
  7. Consolidate card limit/tier/rate actions — single code per event type with
     changeSource: "admin" | "review"
  8. Add OPS_PERMISSION_DENIED for staff/operator forbidden attempts on sensitive
     endpoints
  9. Add COMPANY_INVITATION_REVOKED when/if revoke flow is built
 10. Add OPS_NOTIFICATION_DELIVERY_FAILED audit when customer notification fails

P2 — Noise reduction
 11. Skip ACCOUNT_INTEREST_ACCRUED from Discord — keep ACCOUNT_INTEREST_BATCH_RUN
     summary only
 12. Remove dead code — staff-audit-events.ts, unused DISCORD_SKIP_ACTIONS entries
 13. Complete ACTION_LABELS for deal room, Alta Card, relationship, and cron actions
 14. Review attachment download auditing — uploads yes; downloads may be excessive

P3 — Completeness / future-proofing
 15. Interbank execution audit when wire fulfillment workflow exists
 16. ALTA_PRIVATE_BANKER_ASSIGNED when per-client banker is stored in DB
 17. COMPANY_BUSINESS_ACCESS_CHANGED for module-level permission changes
 18. Unified OPS_CRON_JOB_STARTED/COMPLETED/FAILED wrapper around all cron entry points
 19. Bot infrastructure failures — command crashes, internal HTTP 5xx
 20. Introduce action enum / registry in TypeScript to prevent naming drift


================================================================================
SUMMARY
================================================================================

Alta's audit system has a solid foundation: a single write path, rich deal-room
and Alta Card coverage, and sensible Discord noise filtering for CRM/message sync.

Main weaknesses for a court-ready record:
  - Missing failure trails
  - Inconsistent action naming
  - Misattributed bot events
  - Per-account interest accrual potentially spamming staff Discord

Addressing P0–P1 items would materially improve defensibility without adding UI noise.

  - altaweb/src/server/audit.service.ts
  - altaweb/src/lib/staff-audit/audit-log-discord-bridge.ts
  - altaweb/src/lib/internal/audit-metadata.ts
  - altaweb/prisma/schema.prisma (AuditLog model)
  - altaweb/src/server/bot-actions.service.ts
