Alta Bank Admin Panel Audit
============================
Date: 2026-06-25
Scope: /internal and all operator/admin areas
Purpose: Plan rebuild into a simple, dense, operational bank console (Stripe / Mercury / Linear / Retool style)
Status: Audit only — no implementation

EXECUTIVE SUMMARY
-----------------

The internal console mixes three incompatible products:

1. Live operational tooling (bank queues, lending, Alta Card, audit, customer 360) — real DB, mostly workable
2. Marketing-style page chrome (PageShell hero, eyebrow, descriptions, stat cards) — wrong density for ops
3. Mock preview pages (exchange, terminal, IPOs, listings, API apps) — fake data with action buttons

The biggest usability problems are duplicated queues, hub pages that repeat dedicated pages, two action UX patterns (safe OpsConfirmDialog vs one-click BankReviewButton), and no single workspace model (list → detail with audit, notes, related records).


FINDINGS BY SEVERITY
====================

CRITICAL
--------

1. One-click money actions without confirmation or reason
   - BankReviewButton on deposits, withdrawals, account freeze/close, lending approve/deny/freeze/waive, company verify, Alta Card status changes
   - Operator error risk; weak audit trail (reviewNote usually omitted)
   - Files: src/components/bank/bank-review-button.tsx, most internal queue pages

2. Same queues rendered in 2–3 places
   - /internal/bank embeds pending deposits, withdrawals, account openings
   - Also /internal/bank/deposits, /withdrawals
   - Dedicated deposits page has bulk approve + filter; hub does not

3. Mock data mixed with live ops
   - Routes: /internal/exchange, /terminal, /ipos, /listings, /api-applications
   - Mock transfers section on /internal/bank
   - Mock feature flags on /internal/settings
   - Companies DB fallback to @/lib/internal/api on loader failure

4. Customer marketing shell on every admin page
   - InternalPageShell → PageShell with eyebrow, large title, description, hero animation
   - Wastes vertical space; feels like marketing site, not ops console

5. No admin vs operator route separation
   - canAccessInternal = admin OR operator; same nav and pages for both
   - Operators reach maintenance UI, tag grants, manual interest triggers (some blocked server-side only)

6. Global search statement links go to customer route
   - ops-global-search.service.ts → /bank/statements/${id}
   - Operators leave admin context on search


HIGH
----

1. /internal/bank/transfers ≈ /internal/bank/scheduled
   - Both load fetchInternalScheduledTransfers(); overlapping panels

2. Dual customer workspaces
   - /internal/users/$userId vs /internal/relationships/$userId
   - Company detail vs /companies/$companyId/relationship

3. Lending index is a wall of inline loan cards
   - /internal/lending renders full InternalActiveLoanCard for every active loan
   - Loan workspace exists at /loans/$loanId but queue doesn't link cleanly

4. Alta Card hub duplicates sub-routes
   - /internal/alta-card embeds applications + cards
   - Also /applications, /reviews

5. Audit missing on high-risk workspaces
   - Loan detail, Alta Card detail, bank hub queues — no entity audit panel

6. Activity feed ≠ audit log
   - Dashboard "Activity feed" vs /internal/audit

7. No centralized system jobs view
   - Cron split across API routes
   - Alta Card scheduler panel only on Alta Card hub
   - Bank statement scheduler on statements page
   - Lending batch buttons on lending index

8. Inconsistent confirm UX
   - OpsConfirmDialog (good) vs BankReviewButton vs window.confirm / window.prompt

9. Deal room URLs redirect to lending
   - /internal/lending/deal-rooms/* → redirect to /internal/lending

10. Deposits route double-wraps shell
    - InternalDepositsQueue renders its own InternalPageShell inside route


MEDIUM
------

1. Nav too wide; hidden sub-routes (scheduled, interest, Alta Card apps/reviews/card detail not in nav)
2. Lending nav highlight breaks on detail (exact: true on Lending link)
3. Status labels inconsistent (StatusBadge maps free-form strings; domain enums differ per product)
4. Reports page is stat cards only — no drill-down
5. Compliance is signal dashboard only — no cases; overlaps Exceptions + Dashboard
6. Private banking has no dedicated queue (privateReview search param on user page only)
7. Global search gaps (no Alta Card, application, review, company relationship, cron job)
8. Tables often lack row → workspace links
9. Settings feature flags are fake but presented as Active/Suspended
10. User tag/status UI says audit "planned" but server already audits
11. Loan freeze may skip audit log (freezeLoan / unfreezeLoan in loan.service)
12. Interbank transfers section is mock on otherwise live bank ops page


LOW
---

1. Legacy /bank/admin/* redirects (→ /internal/users, /internal/bank)
2. Embeds page in Tools — niche, OK
3. Notifications bell — deal-room only, not documented in nav
4. Duplicate global search on dashboard (hideSearch + inline duplicate)
5. VITE_UI_LAB_MODE auth bypass — dangerous if misconfigured in prod
