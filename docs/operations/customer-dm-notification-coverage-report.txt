Alta Customer DM Notification Coverage Report
==============================================
Scope: Alta web app (altaweb) + Alta Companion Discord Bot (altabot)
Date: July 3, 2026

Architecture
------------
Customer DMs are sent via Discord REST API (discord-dm.service.ts). The bot
exposes internal HTTP fallback endpoints; hub banking UI uses interaction replies,
not DMs.

Delivery paths:
  1. Standard — createUserNotification -> in-app row + optional DM (respects prefs)
  2. Operator — deliverOperatorCustomerNotification -> DM only, supports silent
     mode, bypasses user prefs
  3. Direct — invitations + Secure Deal Room opened (bypass dispatch/prefs in some
     cases)

6. DMS THAT SHOULD SUPPORT SILENT MODE
================================================================================

Silent mode (silentNotification: true) is implemented for operator actions only
via deliverOperatorCustomerNotification.

Should support silent (already do)
----------------------------------
All 19 operator kinds: manual credit/debit, fees, reversals, corrections,
freeze/unfreeze, close/reopen, holds, restrictions, transaction review,
payment blocked/reversed.

Should support silent (not yet)
-------------------------------
  - Deal room stakeholder notifications (internal doc uploads)
  - Loan/card application denied (sensitive — staff may want in-app only)
  - Scheduled transfer failure (ops may fix before customer knows)

Should NOT support silent
-------------------------
  - Customer-initiated submissions (deposit, withdrawal, transfer)
  - Invitations (must reach invitee)
  - Alta Pay sent/received (both parties need record)
  - Security / freeze — silent off by default; require explicit staff choice with
    audit trail


Silent mode verification (current behavior)
-------------------------------------------
Requirement                              Status
---------------------------------------  ------------------------------------------
Default is notify customer               Yes — shouldNotifyCustomer() true unless
                                           silentNotification === true
Silent skips customer DM                 Yes
Silent still creates audit log           Yes — audit written with metadata regardless
Silent still sends staff audit Discord   Yes — "Silent — customer not notified"
Silent recorded in audit metadata        Yes — silentNotification,
                                           customerNotificationSent,
                                           silentNotificationChosenByUserId

Gap: Silent toggle only on operator UI panels — not on loan/card approve/deny
(which use standard notifications without silent option).

================================================================================
SUMMARY
================================================================================

Alta has a solid foundation for customer DMs on core banking submissions,
approvals, transfers, Alta Pay, operator account actions (with silent mode),
invitations, and deal room milestones.

Biggest gaps:
  - Failure notifications (transfer, pay, scheduled transfer)
  - Loan/card payment lifecycle
  - Secure Deal Room staff messages (in-app only today)
  - Scheduled transfer outcomes
  - Unreliable delivery tracking/retry
