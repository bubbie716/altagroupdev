ALTA GROUP FOOTER AUDIT
=======================

Scope: All routes, shared layouts, auth/maintenance pages, bottom CTAs, legal
lines, and page-ending sections.

Date: June 22, 2026
Status: Audit only — no code changes implemented.


EXECUTIVE SUMMARY
-----------------

The site has one real global footer (SiteFooter in src/components/site-nav.tsx),
rendered on ~80 routes via PageShell or the governance layout. Internal admin
(~30 routes) has no footer. Auth uses a separate LoginPortalFooter on /login
only; /maintenance and /access-restricted have no footer at all.

Main problems:

1. Same heavy marketing footer on homepage and bank dashboard — full 4-column
   sitemap + legal on every authenticated platform page.
2. Login footer copy reads like Alta Private ("Discretion assured · Relationship
   managed") on a public sign-in page.
3. Inconsistent auth-adjacent pages — login has a micro-footer; maintenance and
   access-restricted do not.
4. Alta Private page adds a custom invitation footnote, then still shows the full
   public SiteFooter underneath.
5. Stub links — Press, Careers, and NCC all route to /governance.
6. Statement PDFs end with transaction tables and no legal/disclaimer footer.

  Private page — long scroll + custom footnote + mt-32 footer gap

  404 / error — no footer

3. BRAND / MESSAGE ISSUES
-------------------------

Current structure (target):
  Alta Group N.V. = parent
  Alta Bank = banking division (public)
  Alta Exchange = exchange / capital markets (public)
  Alta Terminal = product inside Alta Exchange
  NCC = planned clearing/settlement

Conflicts found:
  Login footer names NCC as co-copyright entity
  Login footer uses private-banking tone on public auth
  Home divisions vs footer — NCC listed as live division alongside Bank/Exchange
  Terminal in footer Platform column without Exchange parent context
  "Settlement T+0" in global footer implies live clearing
  Governance content says NCC "In Development" — footer does not

What's correct:
  Alta Bank / Exchange links in footer
  Alta Private invitation language confined to /bank/private and private products
  Governance pages use same SiteFooter as public site
  Internal pages don't imply invitation-only access

5. RECOMMENDED FOOTER SYSTEM
----------------------------

A. Global public footer (SiteFooter)
   Use on: Home, Governance, About, Leadership, Documents, unauthenticated marketing
   Contents: Wordmark, tagline, Divisions, Company links (real pages only), full legal
   Remove stub links until pages exist (Press, Careers)

B. Platform footer (PlatformFooter — new, slim)
   Use on: Authenticated Bank, Exchange, Terminal, Companies, Profile
   Contents: © 2026 Alta Group N.V. · division name; data disclaimer; 3–4 inline links
   Drop mt-32 or reduce to mt-16 on app pages

C. Internal footer (InternalMicroFooter — new, minimal)
   Use on: All /internal/* routes
   Contents: Alta Internal · Operator Console; © 2026 · Authorized personnel only
   Height: ~48px bar — no columns

D. Legal micro-footer (LegalMicroFooter — refactor LoginPortalFooter)
   Use on: Login, maintenance, access-restricted, 404/error
   Contents: © 2026 Alta Group N.V.; infrastructure line; no Discretion assured

E. Document footer (StatementFooter — new)
   Use on: Statement print/PDF only
   Contents: Alta Bank legal name, support line, timestamp, confidentiality notice

F. Page-specific footnotes (keep, don't replace global footer)
   /bank/private — invitation footnote; swap SiteFooter for slim platform footer
   / — consider shortening ClosingCTA vs full footer duplication

CONCLUSION
----------

The site needs a tiered footer system: full public footer for marketing/governance,
slim platform footer for authenticated app pages, minimal internal bar for ops,
and corrected legal micro-footer for auth edge cases. The highest-impact fixes are
LoginPortalFooter copy, stub link removal, Alta Private footer suppression, and
adding footers to maintenance/access-restricted pages.
