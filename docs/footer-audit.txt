Alta Ecosystem Footer Audit
===========================
Date: July 6, 2026

Goal
----
Every Alta website should feel like an independent product while still feeling like
part of the same financial ecosystem. The footer is one of the primary ways users
discover the rest of the ecosystem.

Sites audited
-------------
- Alta Group (altagroup.dev)
- Alta Bank (bank.altagroup.dev)
- Alta Exchange (exchange.altagroup.dev)
- Alta Terminal (terminal.altagroup.dev)
- Status (status.altagroup.dev — external redirect)
- Docs (docs.altagroup.dev — coming soon via /docs)


1. FOOTER COMPONENTS FOUND
--------------------------

Component              File                                      Used by
---------              ----                                      -------
SiteFooter             src/components/footers.tsx                  Variant router: marketing / dashboard / auth / legal / none
MarketingFooter        src/components/footers.tsx                Public/marketing pages (corporate, bank, exchange, terminal)
DashboardFooter        src/components/footers.tsx                Authenticated app pages (/bank/*, /exchange/*, etc.)
AuthenticationFooter   src/components/footers.tsx                /, /login, /maintenance, /access-restricted
CheckoutLegalLinks     src/components/footers.tsx                Merchant checkout (/pay/$slug) — inline only, not site chrome
Footer variant resolver src/lib/platform/footer-variant.ts       Path -> variant mapping
Link config            src/lib/site/site-links.ts                Ecosystem, support, corporate section links
Legal doc config       src/lib/legal/legal-document-registry.ts  Entity sections, compact docs, disclaimers

Not footers (excluded from audit):
- DialogFooter (UI primitives)
- Discord embed footer fields
- invoiceFooterText / paymentLinkFooterText (merchant branding)
- Deprecated PageShell hideFooter prop (ignored; footer mounts at root only)
