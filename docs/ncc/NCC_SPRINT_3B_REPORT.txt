3–23. Architecture summary

See docs/ncc/NCC_SPRINT_3B_REPORT.md and Sprint 3B.1 report for full detail.
Key post-remediation facts:

- Credential format: ncc_(live|test)_<hexPrefix>_<secret>
- Webhook delivery connects only to a DNS-validated pinned public IP
- DELIVERING / PROCESSING claims use claimedAt + claimToken leases
- Settlement list validation returns stable 4xx (no Prisma 500s from bad enums)
- Audits written for credential expiry and terminal webhook success/failure
- Production requires NCC_SECRETS_KEY (min 32); no hardcoded pepper in production
