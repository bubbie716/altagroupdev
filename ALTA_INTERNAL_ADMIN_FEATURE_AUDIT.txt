Alta Internal Admin Panel — Feature Audit

Scope: /internal and all bank/user/company/lending admin routes
Focus: Missing admin functionality — can staff operate without DB edits?
Date: June 22, 2026


================================================================================
ROLE SUMMARY
================================================================================

Operator: All /internal routes; most bank/lending/company ops; cannot grant
admin/operator tags; cannot freeze/reactivate users; cannot overdraft-debit;
cannot run scheduled deposit interest accrual or apply manual interest.

Admin: Everything operators can do, plus staff tags, full user status control,
overdraft debits, deposit interest accrual, manual interest apply.

================================================================================

CRITICAL (launch blockers for controlled beta)

1. Alta Pay admin ledger + refund/reverse — no way to investigate or correct
   merchant payments
2. Transaction reversal workflow — adjustments are poor substitute; no linked
   reversal
3. Admin manual loan payment — cannot post payment on behalf of borrower
4. Automated deposit interest trigger — admin can batch manually but nothing
   schedules it
5. Pagination on users, accounts, audit — staff cannot see full population
6. Alta Pay payment search — disputes require hunting transactions per account


HIGH

7. User bank account -> account admin deep links
8. Company bank accounts / loans / payments from company page
9. Admin company member management
10. Company suspend / reactivate
11. Loan repayment schedule in admin UI
12. Editable loan term at approval
13. Bulk queue actions (deposits, withdrawals, loans)
14. Review note field on deposit/withdrawal approve/deny
15. Negative balance / exception dashboard
16. Compliance case workflow
17. Audit log export
18. Cron run history / health
19. Failed transaction unified queue
20. Operator cannot reactivate frozen users (admin-only today)

24. Reopen closed account

================================================================================
LAUNCH BLOCKER ADMIN FEATURES (CONTROLLED BETA ONLY)
================================================================================

For a small controlled beta (hundreds of users, few operators), staff can run
core flows. These gaps still block a confident beta:

1. Alta Pay admin visibility + correction path
   Merchant payments are live; no ops tooling

2. Manual loan payment (admin)
   Borrower unavailable = stuck loan

3. Deposit interest automation or documented daily admin runbook
   Savings products won't pay without manual batch

4. Transaction correction beyond blind adjustment
   Error recovery is error-prone

5. User/account deep navigation
   Daily ops friction causes mistakes

Pagination and bulk ops matter less at beta scale but become blockers quickly
after.


================================================================================
CAN RUN BANK WITHOUT DB EDITS?
================================================================================

VERDICT: PARTIALLY YES — FOR A SMALL CONTROLLED BETA ONLY


YES, WITHOUT DB EDITS, STAFF CAN:

- Review and approve/deny deposits (with proof) and withdrawals
- Approve account openings; freeze, unfreeze, close accounts
- Credit/debit via admin adjustments (with reason + audit)
- Run manual interest campaigns and (admin) scheduled deposit interest batches
- Manage scheduled transfers and payroll execution
- Generate statement batches and view statements
- Review, approve, deny, disburse, service, freeze, adjust, and mark paid off
  loans
- Manage users (tags, status within role limits), notes, and audit
- Verify/reject companies
- Monitor compliance signals and platform vitals


NO — THESE STILL REQUIRE DB EDITS OR UNSUPPORTED WORKAROUNDS:

Task                          Workaround without DB          Gap
----------------------------  -----------------------------  -------------------------
Refund Alta Pay               Offsetting adjustments         No reference link/audit
Reverse specific transaction  Opposite adjustment            No tie to original txn
Void statement                —                              No API
Suspend company               —                              No API
Assign ticker (admin)         —                              User-only settings
Change loan term at approval  —                              Fixed from application
Post loan payment for user    —                              No admin payment UI
Reactivate user (operator)    —                              Admin only
Reopen closed account         —                              No API
Compliance case               Audit + notes                  No case model
Deal room lifecycle           —                              Mock only
Feature flags / maintenance   —                              Not persisted
Interbank scheduled execution —                              Stuck PENDING_REVIEW


BOTTOM LINE:

Core retail banking (deposits, withdrawals, accounts, adjustments, lending
lifecycle) is operable from the panel.

Alta Pay, merchant ops, precise reversals, compliance cases, and several
company/loan edge cases are NOT.

================================================================================
BRUTALLY HONEST SUMMARY
================================================================================

The Internal Admin Panel is REAL and USABLE for bread-and-butter work: queues,
account lifecycle, adjustments, lending approval/servicing, user tagging, and
audit.

It is NOT a complete operations platform.

The largest functional holes are Alta Pay (almost no admin surface), payment/
transaction reversal, admin-initiated loan payments, and company-level
operations.

Several backend capabilities exist but are NOT exposed in UI (statement void,
company/loan notes, single-account statements, audit date filters).

For a controlled beta with hands-on admins and low volume: YES, you can run the
bank from the panel with occasional adjustment workarounds.

For institution-scale operations without database access: NO — staff will hit
walls on Alta Pay, reversals, compliance, and company/merchant workflows within
the first week.
