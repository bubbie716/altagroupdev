=== Platform Overview

--- Project purpose

Alta Capital Suite is a TanStack Start (Vite + React 19 + SSR) web application for the fictional Alta Group financial infrastructure in the Newport roleplay economy. It presents four product divisions — Alta Bank, Alta Terminal, Alta Exchange, and NCC (Newport Clearing Corporation) — plus a corporate marketing site, Discord authentication, and an internal admin ops portal.

Status: Partial — Strong marketing/preview UI across products; real persistence exists only for auth identity (users, sessions, company registry, tags). All financial/trading/banking operations are simulated.

--- Major divisions

| Division | Route prefix | Status |
|----------|--------------|--------|
| Alta Group (marketing) | `/`, `/governance` | Partial — static/marketing UI |
| Alta Bank | `/bank/*` | Partial — mock banking UI |
| Alta Terminal | `/terminal/*` | Partial — mock brokerage UI |
| Alta Exchange | `/exchange/*` | Partial — mock market UI + docs |
| NCC | No dedicated routes | Missing — described only on `/governance` and in copy |
| Internal admin | `/internal/*` | Partial — real admin auth, mock ops data |
| Auth & profile | `/login`, `/profile` | Partial — real OAuth + DB sessions |

--- Routing structure

Framework: TanStack Router file-based routes (`src/routes/`), auto-generated `routeTree.gen.ts`.

Layout patterns:
- Exchange & Terminal: parent `route.tsx` + `<Outlet />` for nested pages
- Bank: flat route files (no `/bank/route.tsx` layout)
- Internal: parent `/internal/route.tsx` with admin guard
- Company profiles: `/exchange/company/$ticker/route.tsx` → `index.tsx` + `owner.tsx`

Legacy redirects:
- `/dashboard` → `/terminal`
- `/markets` → `/exchange`
- `/bank/admin/*` → `/internal/*` (clients → users, loans/private → bank)

Total route files: ~54 `.tsx` route components + 2 API auth routes.

Not in DB: Bank accounts, balances, transfers, trades, orders, listings (beyond seed companies), IPO applications, API keys, compliance cases, market data.

=== Build Roadmap

--- 1. Critical Next Steps

1. Define real data boundaries — Decide which entities move from mock to Postgres first (likely user-linked bank account + holdings).
2. Enforce access tags — Gate `/bank/private` on `private_client` tag; gate API docs on `developerAccessStatus` or API approval.
3. Remove or secure sensitive mock exposure — Leaderboard and private banking pages visible without appropriate auth.
4. Legal baseline — Publish simulation disclaimer + Terms + Privacy pages (even minimal) before wider access.
5. Admin mutation path — At least one real admin action (e.g. grant tag, freeze user) persisted to DB.
6. Unify market data source — Single stock/index universe (fix NSX/AEX/PRTH inconsistencies).

--- 2. Before Beta Launch

1. Postgres models for bank accounts, holdings, orders (read path minimum).
2. Replace mock dashboard data with user-scoped queries for Bank + Terminal.
3. Working transfer submission (even if simulated settlement backend).
4. Exchange listing application persistence + internal review workflow.
5. Server-side API key storage (replace localStorage).
6. Issuer portal authentication (server-side, tied to company membership).
7. Internal admin: search, pagination, functional approve/reject for one queue (IPO or API).
8. Email or notification on application status changes.
9. Session management page (view/revoke sessions).
10. Error/loading states (React Query hooks when APIs go async).

--- 3. Before First Real Users

1. Full auth audit trail (login, logout, tag changes, admin actions).
2. Bank ledger with immutable entries; transfer idempotency.
3. Terminal order entry with validation and audit (paper trading mode acceptable).
4. Exchange market data API (HTTP) with documented rate limits.
5. Real IPO subscription flow with allocations.
6. Company membership enforcement on issuer and company-scoped bank features.
7. Compliance case linkage to real users/transactions.
8. NCC minimum viable: transfer settlement status visibility.
9. Production monitoring, backup, and migration strategy.
10. Remove dev mock overrides from production builds.
